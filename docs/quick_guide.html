Quick Guide — Como atualizar o site

Guia passo a passo para quando você quiser mexer no site (ex: adicionar um post novo no blog, editar a página inicial, trocar o CV, etc.). Este arquivo NÃO sobe para o GitHub (está no .gitignore) — é só uma cola local para você.

0. Conceito básico (leia uma vez, depois pode pular)

Este site é feito em Quarto. Você escreve arquivos-fonte .qmd (texto + código R), e o Quarto renderiza (compila) isso em HTML dentro da pasta docs/. É a pasta docs/ que o GitHub Pages efetivamente publica — não os .qmd.

Ou seja: editar o .qmd sozinho não muda o site publicado. Você precisa sempre rodar quarto render antes de subir (git push), senão o docs/ fica desatualizado em relação ao que você escreveu.

Fluxo resumido sempre que for atualizar algo:

editar .qmd  →  quarto render  →  git add/commit/push

1. Adicionar um post novo no blog (exemplo completo)

1.1 Copie o template

Dentro da pasta blog/, existe 0000-template.qmd — é o esqueleto de um post. Copie ele para um arquivo novo, com um número sequencial e um nome curto do tema (isso controla a ordem/organização dos arquivos):

Copy-Item blog\0000-template.qmd blog\0001-inflacao-brasil.qmd

Exemplos de nomes: 0001-inflacao-brasil.qmd, 0002-taylor-rule.qmd, etc.

⚠️ Não edite o 0000-template.qmd original — ele fica como modelo para o próximo post. Sempre copie para um arquivo novo.

⚠️ Atenção: hoje o 0000-template.qmd está sendo publicado como se fosse um post de verdade (aparece na listagem do blog). Isso é um bug pendente — combine comigo depois para excluir esse arquivo da listagem, ou simplesmente não deixe conteúdo de rascunho nele.

1.2 Edite o cabeçalho (front matter)

Abra o arquivo novo (blog/0001-inflacao-brasil.qmd) e edite as primeiras linhas:

---
title: "Título real do post"
description: "Uma frase resumindo o que o post mostra."
date: 2026-07-07
categories: [inflação, brasil, forecasting]
---
  • date: use o formato AAAA-MM-DD. É essa data que ordena os posts na listagem (mais recente primeiro).
  • categories: tags livres, aparecem como filtro na página do blog.

1.3 Escreva o conteúdo

Substitua as seções TODO do template pelo conteúdo real: motivação, dados (idealmente puxando de fonte pública via API, não dado sintético), método, código R, gráfico, conclusão. Apague os comentários <!-- TODO ... --> e o callout de aviso (🚧) no topo quando terminar.

1.4 Pré-visualize localmente (opcional, mas recomendado)

Antes de publicar de verdade, veja como o post fica:

quarto preview

Isso abre um servidor local (algo como http://localhost:XXXX) que atualiza sozinho enquanto você edita. Ctrl+C no terminal para encerrar.

1.5 Renderize o site inteiro

Quando estiver satisfeito, gere o HTML final:

quarto render

Isso atualiza a pasta docs/ inteira (não só o post novo) — inclusive a listagem do blog, sitemap, etc. Sempre rode isso antes de dar push.

1.6 Suba para o GitHub

git add .
git commit -m "Add post: inflação no Brasil"
git push

1.7 Confira no ar

Espere ~1 minuto e acesse https://raphaludwig.github.io/blog/ — o post novo deve aparecer na listagem.

2. Editar uma página existente (ex: Home, CV, About)

Mesma lógica, só que sem copiar template:

  1. Edite o .qmd da página (ex: index.qmd para a home).
  2. quarto render
  3. git add . && git commit -m "..." && git push

3. Trocar o PDF do currículo

  1. Substitua o arquivo em cv/Raphael_Ludwig_CV.pdf pelo novo PDF (mesmo nome de arquivo).
  2. quarto render (ele está listado em resources no _quarto.yml, então precisa re-renderizar para copiar pro docs/).
  3. git add . && git commit -m "Update CV" && git push

4. Erros comuns

Sintoma Causa provável Solução
Editei o .qmd mas o site no ar não mudou Esqueceu de rodar quarto render antes do push Rode quarto render, depois git add/commit/push de novo
quarto render não recalcula um gráfico/código que mudei Cache de freeze: auto (definido no _quarto.yml e blog/_metadata.yml) Apague a pasta _freeze/ daquele arquivo específico, ou rode quarto render --execute
Post novo não aparece na listagem do blog Data (date:) no futuro, ou front matter mal formatado (YAML quebrado) Confira o date: e a sintaxe do cabeçalho ---
quarto: comando não encontrado Quarto não está instalado / não está no PATH Reinstale ou reabra o terminal

5. Checklist rápido (cola mental)

[ ] Conteúdo revisado (sem TODOs esquecidos)

[ ] git commit -m "mensagem clara"
